### PostgreSQL Row-Level Security

- **SELECT**: Allow all (no filter â€” apps need to read their drafts)
- **INSERT**: `tenant_id IS NULL OR tenant_id = current_tenant_id()` â€” drafts can be NULL
- **UPDATE**: Same as INSERT â€” approval binds tenant_id

## Status State Machine

```
                 +--> Approved  (user approves + KC registration succeeds)
                 |
  Draft ---------+--> Denied    (user denies)
                 |
                 +--> Failed    (KC returns 409 UUID collision)
                 |
                 +--> Expired   (auto-transition on read when expires_at < now)

  All terminal states are final â€” no further transitions.
```

### Transition Rules

| Transition | Trigger | Fields Set |
|-----------|---------|------------|
| Draft -> Approved | `update_approval()` | `tenant_id`, `user_id`, `approved`, `approved_role`, `access_request_scope`, `status`, `updated_at` |
| Draft -> Denied | `update_denial()` | `user_id`, `status`, `updated_at` |
| Draft -> Failed | `update_failure()` | `error_message`, `status`, `updated_at` |
| Draft -> Expired | `get()` (lazy) | `status`, `updated_at` |

### Validation Guards

- Only Draft requests can be approved/denied/failed
- Non-Draft -> `DbError::AccessRequestNotDraft { id, status }`
- Expired draft -> `DbError::AccessRequestExpired(id)`

| Method | Parameters | Notes |
|--------|-----------|-------|
| `create()` | `&AppAccessRequest` | Uses `with_tenant_txn` with empty tenant for NULL |

## Multi-Tenant Behavior

- **Draft**: `tenant_id = NULL` â€” not bound to any tenant
- **Approved**: `tenant_id = approver's active tenant` â€” bound at approval
- **Denied/Failed/Expired**: `tenant_id` unchanged from previous state
- RLS policies allow INSERT/UPDATE with NULL tenant_id (for drafts)
- `get_by_access_request_scope()` requires tenant_id for scoped lookups
- Unique index ensures one `access_request_scope` per tenant
