# Per-Group CORS for Session vs Non-Session APIs

## Context

Session-only APIs (`user_session_apis`, `power_user_session_apis`, `manager_session_apis`, `admin_session_apis`) currently share a single permissive `CorsLayer` with all other routes. While session auth already has strong protections (SameSite=Strict cookies, Sec-Fetch-Site check), adding restrictive CORS provides defense-in-depth by preventing cross-origin preflight from succeeding — browsers won't even send the actual request to session endpoints.

**Outcome**: Session-only endpoints block cross-origin requests at the CORS preflight stage. Non-session endpoints (OpenAI-compatible, OAuth, public) retain permissive CORS for external tool/app access.

1. `crates/routes_app/src/routes.rs` — Core route restructuring + CORS layers

## Implementation

### Step 1: Add CORS helper functions to `routes.rs`

```rust
fn permissive_cors() -> CorsLayer {
    CorsLayer::new()
        .allow_origin(Any)
        .allow_methods(Any)
        .allow_headers(Any)
        .allow_private_network(true)
        .allow_credentials(false)
}

fn restrictive_cors() -> CorsLayer {
    CorsLayer::new()
    // Defaults: no origins, no methods, no headers, no private network
    // OPTIONS gets 200 but no Access-Control-Allow-Origin → browser blocks
}
```

### Step 3: Restructure router composition

Replace the current single `protected_apis` + global CorsLayer (lines 467-505) with:

```rust
// Session-protected routes (RESTRICTIVE CORS — blocks all cross-origin)
let session_protected = Router::new()
    .merge(user_session_apis)            // modified: without overlapping routes
    .merge(power_user_session_apis)
    .merge(admin_session_apis)
    .merge(manager_session_apis)
    .route_layer(from_fn_with_state(state.clone(), auth_middleware))
    .layer(restrictive_cors());

// API-protected routes (PERMISSIVE CORS — external tools/apps need access)
let api_protected = Router::new()
    .merge(user_apis)
    .merge(user_oauth_apis)
    .merge(overlapping_session_apis)     // session auth but permissive CORS
    .merge(toolset_exec_apis)
    .merge(mcp_exec_apis)
    .merge(power_user_apis)
    .route_layer(from_fn_with_state(state.clone(), auth_middleware))
    .layer(permissive_cors());

// Public + optional auth (PERMISSIVE CORS)
let public_router = Router::new()
    .merge(public_apis)
    .merge(optional_auth)
    .layer(permissive_cors());

// Final router — NO global CorsLayer
let router = Router::<Arc<dyn AppService>>::new()
    .merge(public_router)
    .merge(session_protected)
    .merge(api_protected)
    .merge(SwaggerUi::new("/swagger-ui").url("/api-docs/openapi.json", openapi))
    .with_state(state);
```

**Key design notes:**
- `.route_layer(auth_middleware)` is INSIDE `.layer(cors)` — CORS intercepts OPTIONS before auth runs (no 401 on preflight)
- `auth_middleware` is duplicated across groups — it's a stateless function with `Arc<dyn AppService>`, no behavioral difference
- Zero cross-group path overlaps after extraction — Axum merge is safe
- SwaggerUI may need `.layer(permissive_cors())` if cross-origin API docs access is required; test and adjust

pub fn cors_preflight_request(path: &str, method: &str, origin: &str) -> Request<Body> {
    Request::builder()
        .method("OPTIONS")
        .uri(path)
        .header("Origin", origin)
        .header("Access-Control-Request-Method", method)
        .header("Host", "localhost:1135")
        .body(Body::empty())
        .unwrap()
}

Add section:
```markdown
## Overlapping CORS path structure

- **Currently**: 3 URL paths have both session-only and non-session methods, preventing per-method CORS separation:
  - `/bodhi/v1/toolsets`: POST (session) + GET (oauth)
  - `/bodhi/v1/mcps`: POST (session) + GET (oauth)
  - `/bodhi/v1/mcps/{id}`: PUT/DELETE (session) + GET (exec/oauth)
- **Workaround**: Session methods at overlapping paths are in the permissive CORS group (auth still enforces session-only access)
- **Future**: Move external app endpoints to `/bodhi/v1/apps/{toolsets,mcps}`, making `/bodhi/v1/{toolsets,mcps}` fully session-only with restrictive CORS
```
