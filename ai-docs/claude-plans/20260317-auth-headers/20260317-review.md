### Test: `single header auth - wrong credentials → fetch tools fails`

Pattern follows existing negative tests (missing/partial credentials):

1. Create MCP server + auth config via API (header param: `X-Auth-Key`)
2. Navigate to new MCP form
3. Select server, select auth config
4. Fill credential value with **wrong** value (e.g., `wrong-value-123` instead of the expected value)
5. Click Fetch Tools
6. Assert: `expectToolsLoadingHidden()` → `expectToolsListNotVisible()` → `expectToolsEmptyState()`

The mock server's `validateAuth()` checks exact header values. Wrong value → 401 → fetch tools fails → empty state shown.
