# Official SDK Compatibility E2E Tests

## Context

BodhiApp exposes provider-compatible proxy endpoints on three surfaces:

- **OpenAI** / **OpenAI Responses** on `/v1/*` (chat completions, responses, models, embeddings)
- **Anthropic Messages** on `/anthropic/v1/*` (messages, models)
- **Gemini** on `/v1beta/*` (generateContent, streamGenerateContent, embedContent, models)

We already have `crates/lib_bodhiserver/tests-js/specs/api-models/api-live-upstream.spec.mjs` that validates these endpoints via **raw `fetch`** calls. That test catches middleware regressions (x-api-key rewriting, universal `/v1/chat/completions` routing, chat UI wiring) but does **not** verify that the endpoints behave correctly when driven by the provider's **official SDK clients** — which is what real 3rd-party integrations use.

**Goal.** Add a new parameterized E2E spec that drives BodhiApp's proxy endpoints with the official SDKs (`openai`, `@anthropic-ai/sdk`, `@google/genai`), covering every `(format × auth method × capability)` combination as an **independently reported** Playwright test. A failure in one combo must not hide failures in the others. This protects SDK compatibility as an explicit contract.

**User decisions captured up front:**
- Coexist with `api-live-upstream.spec.mjs`; add a header comment there clarifying "direct REST, not SDK".
- Do not create an `anthropic_oauth` alias — Anthropic's public SDK does not officially support the Claude-Code OAuth flow.
- Gemini SDK: `@google/genai` (the new unified SDK, supports `httpOptions.baseUrl` + `apiVersion: 'v1beta'`).
- Capability scope per combo: non-streaming chat, streaming chat, tool use, models list, embeddings (where the format supports it).
- Each combo is its own `test()` block inside a `serial` `describe` — Playwright report shows a row per combo.

## Implementation

### Files to CREATE

**`crates/lib_bodhiserver/tests-js/specs/api-models/api-sdk-compat.spec.mjs`** — the new parameterized spec. Outline:

```js
// SDK-compatibility suite. Unlike api-live-upstream.spec.mjs (raw fetch),
// this file drives each BodhiApp proxy endpoint through the provider's
// OFFICIAL SDK client to guarantee 3rd-party-integration compatibility.
import { test, expect } from '@/fixtures.mjs';
import { ApiModelFixtures } from '@/fixtures/apiModelFixtures.mjs';
import { buildSdkAdapter } from '@/utils/sdk-adapters.mjs';
// page objects + auth-server-client imports as in api-live-upstream.spec.mjs

test.describe.configure({ mode: 'serial' });

test.describe('SDK compatibility — provider endpoints × auth tokens', () => {
  // Suppress per-test DB reset so beforeAll state (models, tokens) survives.
  test.use({ autoResetDb: async ({}, use) => { await use(); } });

  const FORMATS = ['openai', 'openai_responses', 'anthropic', 'gemini'];
  const AUTH_METHODS = ['apiToken', 'appToken'];
  const state = {
    apiToken: null, appToken: null,
    models: {}, // formatKey -> { modelId, effectiveModel }
    embeddingAliases: {}, // formatKey -> { modelId, effectiveModel } for openai, gemini
  };

  test.beforeAll(({ browser: _b }) => {
    for (const f of FORMATS) {
      const k = ApiModelFixtures.API_FORMATS[f].envKey;
      if (!process.env[k]) throw new Error(`${k} not set`);
    }
  });

  test('[setup] login, provision aliases, mint tokens', async ({ page, sharedServerUrl }) => {
    // 1. OAuth login as resource admin.
    // 2. For each FORMATS entry: createAllFormatModels() as in api-live-upstream.
    // 3. Extra step: for openai + gemini, register an EMBEDDING-capable alias
    //    (add text-embedding-3-small / text-embedding-004 to the models list,
    //    or create a second alias with prefix 'emb-oai/' / 'emb-gmn/').
    // 4. mintApiToken() -> state.apiToken
    // 5. Full PKCE flow via OAuthTestApp -> state.appToken
  });

  for (const format of FORMATS) {
    for (const auth of AUTH_METHODS) {
      test.describe(`${format} × ${auth}`, () => {
        const token = () => (auth === 'apiToken' ? state.apiToken : state.appToken);
        const adapter = () => buildSdkAdapter(format, sharedServerUrl, token(), state);

        test(`non-streaming chat`, async ({ sharedServerUrl }) => {
          const out = await adapter().chat({ stream: false });
          expect(out.text.toLowerCase()).toContain('tuesday');
        });

        test(`streaming chat`, async ({ sharedServerUrl }) => {
          const out = await adapter().chat({ stream: true });
          expect(out.chunkCount).toBeGreaterThan(0);
          expect(out.text.toLowerCase()).toContain('tuesday');
        });

        test(`tool use (get_weather)`, async ({ sharedServerUrl }) => {
          const out = await adapter().toolCall();
          expect(out.toolName).toBe('get_weather');
          expect(out.toolArgs.location.toLowerCase()).toContain('san francisco');
        });

        test(`models list`, async ({ sharedServerUrl }) => {
          const ids = await adapter().listModels();
          expect(ids).toContain(state.models[format].effectiveModel);
        });

        if (format === 'openai' || format === 'gemini') {
          test(`embeddings`, async ({ sharedServerUrl }) => {
            const out = await adapter().embed('hello world');
            expect(out.dimensions).toBeGreaterThan(0);
            expect(Array.isArray(out.vector)).toBe(true);
          });
        } else {
          test.skip(`embeddings (not supported by ${format})`, () => {});
        }
      });
    }
  }

  test('[teardown] delete provisioned aliases', async ({ page, sharedServerUrl }) => {
    // Delete all model aliases created in setup.
  });
});
```

Rationale for the shape:
- `test.describe.configure({ mode: 'serial' })` + `test.use({ autoResetDb: noop })` keeps the DB intact across tests inside this describe so the 36 combo tests share one provisioned fixture set. The outer `autoResetDb` still fires before the `[setup]` test, giving us a clean slate for this spec.
- `[setup]` and `[teardown]` are *explicit test blocks* (not `beforeAll`/`afterAll`) because Playwright's `beforeAll` doesn't receive a per-test `page` fixture, and all page objects expect a `page`. This matches how `api-live-upstream.spec.mjs` structures its setup inside the main test.
- Each capability test is its own `test()` — the Playwright HTML report will show 40 rows (1 setup + 36 capability tests + skipped embeddings rows + 1 teardown) with independent pass/fail. A failure in `anthropic × appToken : tool use` does not block `gemini × apiToken : streaming chat`.

Per-format wiring notes:

- **openai** — `new OpenAI({ apiKey: token, baseURL: ${serverUrl}/v1 })`.
  - chat: `client.chat.completions.create({ model, messages, stream })`
  - tool: `tools: [{ type: 'function', function: { name: 'get_weather', parameters: {...} }}]`, assert `choices[0].message.tool_calls[0].function.name`.
  - models: `client.models.list()`.
  - embed: `client.embeddings.create({ model: 'text-embedding-3-small', input })` against the embedding alias.

- **openai_responses** — same OpenAI client, different API surface.
  - chat: `client.responses.create({ model, input: question, stream })`.
  - streaming: iterate `stream.event_stream()` / `for await` yield (SDK v4+).
  - tool: `client.responses.create({ model, tools: [{type:'function', name, parameters}], input })`; assert an `output[i].type === 'function_call'` item.
  - models: `client.models.list()` (same endpoint).
  - embed: `test.skip` — rejected server-side.

- **anthropic** — `new Anthropic({ apiKey: token, baseURL: ${serverUrl}/anthropic })`.
  - SDK sends `x-api-key` by default; our `anthropic_auth_middleware` rewrites to `Authorization: Bearer` — same middleware path the raw `fetchWithApiKey` already exercises, so the OAuth app token (a JWT, not a `bodhiapp_...`) is also accepted here.
  - chat: `client.messages.create({ model, max_tokens: 50, messages, stream })`.
  - streaming: `client.messages.stream(...)` and aggregate text deltas.
  - tool: `tools: [{ name: 'get_weather', input_schema: {...} }]`; assert `response.content` contains a `tool_use` block whose `name === 'get_weather'`; `stop_reason === 'tool_use'`.
  - models: `client.models.list()` (hits `/anthropic/v1/models`).
  - embed: `test.skip` — Anthropic has no public embeddings API.

- **gemini** — `new GoogleGenAI({ apiKey: token, httpOptions: { baseUrl: ${serverUrl}, apiVersion: 'v1beta' } })`.
  - SDK sends `x-goog-api-key`; `gemini_auth_middleware` rewrites to `Authorization: Bearer`.
  - chat non-streaming: `ai.models.generateContent({ model, contents })`.
  - chat streaming: `ai.models.generateContentStream({ model, contents })` — iterate chunks.
  - tool: `tools: [{ functionDeclarations: [{ name: 'get_weather', parameters: {...} }] }]`; assert `response.candidates[0].content.parts[i].functionCall.name === 'get_weather'`.
  - models: `ai.models.list()`.
  - embed: `ai.models.embedContent({ model: 'text-embedding-004', contents: [{ parts: [{ text }] }] })`.

A common `get_weather` schema (JSON Schema, `{ location: string }`) is shared across formats with per-format envelope adjustments — all four providers accept this shape.

## Verification

Work through this sequence during implementation; do not declare the task done until every step passes:

1. **Install deps**: `cd crates/lib_bodhiserver/tests-js && npm install` — confirms the three SDK packages install.
2. **Rust unchanged → no backend rebuild needed.** If `bodhiserver_dev` isn't built yet: `make build.dev-server`.

4. **Run only the new spec first**, standalone project:
   ```bash
   cd crates/lib_bodhiserver/tests-js
   PLAYWRIGHT_TIMEOUT=180000 npx playwright test \
     --project standalone \
     specs/api-models/api-sdk-compat.spec.mjs
   ```

6. **Full suite regression**: `make test.e2e` — ensures the new spec does not disturb the existing `api-live-upstream.spec.mjs` or other specs running in the same worker (serial mode, shared server port 51135).
7. **Multi-tenant project**: the new spec lives under `specs/api-models/` which runs in both projects. Ensure no path-specific assumptions (tenant-aware `sharedServerUrl` is already handled by the fixture).
