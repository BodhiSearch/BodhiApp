## Phase 4 — Pass-through auth rejection + api_format coherence

User directive: bind frontend and backend. `extra_headers` must not contain `Authorization` or `x-api-key`. When `api_format` changes in edit mode, the form must reset and the user must provide a new api_key (backend rejects `Keep`).

### Backend

**New error variants** in `crates/services/src/shared_objs/error_wrappers.rs`:
- `ObjValidationError::ExtraHeadersForbiddenKey(String)` — message: `"Cannot have pass-through authorization headers. '{0}' is not allowed in extra_headers."`
- `ObjValidationError::ApiFormatChangedRequiresNewKey` — message: `"Changing api_format requires a new api_key; cannot keep the existing key."`

**New validator** in `crates/services/src/shared_objs/url_validator.rs`:
- `pub fn validate_extra_headers_no_auth(&serde_json::Value) -> Result<(), ValidationError>` — rejects `Authorization` / `x-api-key` (case-insensitive) in any JSON object value.

**DTO wiring** (`crates/services/src/models/model_objs.rs`): `#[validate(custom(function = "crate::validate_extra_headers_no_auth"))]` on the `extra_headers` field of `ApiModelRequest`, `TestPromptRequest`, `FetchModelsRequest`. Called automatically by `ValidatedJson` in route handlers.

**Defense-in-depth at service layer** (`crates/services/src/models/api_model_service.rs`):
- `validate_extra_headers(&Option<Value>) -> Result<(), ApiModelServiceError>` — service-level mirror, wrapping the validator's check and mapping to `ApiModelServiceError::Validation(ObjValidationError::ExtraHeadersForbiddenKey)`.
- Called at the top of `DefaultApiModelService::create` and `::update`.

**API-format-change coherence** in `DefaultApiModelService::update`:
```rust
if form.api_format != api_alias.api_format && matches!(form.api_key, ApiKeyUpdate::Keep) {
  return Err(ApiModelServiceError::Validation(
    crate::ObjValidationError::ApiFormatChangedRequiresNewKey,
  ));
}
```

### Frontend

**Zod schema** (`crates/bodhi/src/schemas/apiModel.ts`): `validateJsonObjectField` extended to reject `authorization` / `x-api-key` keys (case-insensitive) in `extra_headers`. Error message matches the backend verbatim.

**Form binding** (`crates/bodhi/src/components/api-models/hooks/useApiModelForm.ts`, `handleApiFormatChange`): when the current mode is `edit` and the new `apiFormat` differs from `initialData.api_format`, set `useApiKey = true` (instead of the default `false`). This forces the user to supply a new key, which means the submit path can never generate `ApiKeyUpdate::Keep` after a format change — the backend coherence check becomes unreachable through the UI path.
