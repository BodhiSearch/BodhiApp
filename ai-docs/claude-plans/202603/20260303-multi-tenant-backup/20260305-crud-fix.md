# CRUD Layer Refactor Review Fixes

## Context

The CRUD layer refactor (commit f354b09a0) successfully replaced `WithRejection<Json<...>>` with `ValidatedJson`, renamed `*Form` → `*Request` types, and moved request/response types to services. A post-refactor review identified ~25 fixable issues: redundant validation calls, orphaned code, stale comments, inconsistent parameter naming, a parallel serialization path in `ValidationRejection`, and missing test coverage. This plan addresses all non-informational findings.

## Decisions

- **S-2**: Delegate entire `IntoResponse for ValidationRejection` body to `ApiError::from(self).into_response()` (both arms)
- **Param naming**: Standardize ALL to `request` — `form`, `body`, AND `payload` (only function params, not local test vars)
- **MC-5**: Delete all 8 orphaned standalone validation functions; rewrite tests against `.validate()` framework
- **Toolset validation fns** (`validate_toolset_slug`/`validate_toolset_description`): Also delete after MC-1 removes production callers; rewrite tests
- **Apps handlers**: Add bare `#[derive(Validate)]` + switch to `ValidatedJson` (beyond U-3 rename)
- **OAuth URL tests** (`test_mcp_objs_types.rs`): Delete 3 tests for `validate_oauth_endpoint_url` — no struct-level equivalent exists
