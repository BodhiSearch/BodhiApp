# Table Analysis for Multi-Tenancy

> **Purpose**: Every database table's current schema and the exact changes needed for multi-tenancy.
> Covers migration changes, entity file changes, unique index adjustments, and repository changes.
>
> **Created**: 2026-03-03
> **Update when**: Schema changes during implementation.

---

## Summary

- **Total tables**: 15 (including settings)
- **Tables getting `tenant_id`**: 13 (all except `tenants` itself and `settings`)
- **`tenants` table**: Renamed from `apps`, gets new `id` PK
- **`settings` table**: Stays global (no tenant_id)
- **Migration approach**: Modify existing CREATE TABLE migrations in place

---

## 1. `tenants` table (renamed from `apps`)

**Migration**: `m20250101_000013_apps.rs`

### Current Schema
```sql
apps (
  client_id TEXT PRIMARY KEY,
  encrypted_client_secret TEXT,
  salt_client_secret TEXT,
  nonce_client_secret TEXT,
  app_status TEXT NOT NULL DEFAULT 'setup',
  created_at TIMESTAMP_TZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP_TZ NOT NULL DEFAULT CURRENT_TIMESTAMP
)
```

### New Schema
```sql
tenants (
  id TEXT PRIMARY KEY,                    -- NEW: ULID, generated by app
  client_id TEXT NOT NULL UNIQUE,         -- WAS PK, now unique index
  encrypted_client_secret TEXT,
  salt_client_secret TEXT,
  nonce_client_secret TEXT,
  app_status TEXT NOT NULL DEFAULT 'setup',
  created_at TIMESTAMP_TZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP_TZ NOT NULL DEFAULT CURRENT_TIMESTAMP
)
CREATE UNIQUE INDEX idx_tenants_client_id ON tenants(client_id);
```

### Entity Changes
- File: `crates/services/src/apps/app_instance_entity.rs` → `tenant_entity.rs`
- table_name: `"apps"` → `"tenants"`
- Add `id: String` as PK (`#[sea_orm(primary_key, auto_increment = false)]`)
- `client_id` becomes regular column (not PK)

### Domain Object Changes
- `AppInstance` → `Tenant` struct, add `id: String` field
- `AppStatus` stays as-is

## Repository Changes Summary

Every repository trait that does CRUD operations needs `tenant_id: &str` parameter added:

| Repository Trait | File | Key Methods to Update |
|---|---|---|
| `TenantRepository` | `crates/services/src/apps/tenant_repository.rs` | New: `get_tenant_by_client_id()` |
| `TokenRepository` | `crates/services/src/tokens/token_repository.rs` | list, get, create, update, delete |
| `ToolsetRepository` | `crates/services/src/toolsets/` | list, get, create, update, delete, get_config, set_config |
| `McpServerRepository` | `crates/services/src/mcps/mcp_server_repository.rs` | list, get, create, update, delete |
| `McpInstanceRepository` | `crates/services/src/mcps/mcp_instance_repository.rs` | list, get, create, update, delete |
| `McpAuthRepository` | `crates/services/src/mcps/mcp_auth_repository.rs` | list, get, create, delete |
| `DownloadRepository` | `crates/services/src/models/` | list, get, create, update |
| `ApiAliasRepository` | `crates/services/src/models/` | list, get, create, update, delete |
| `ModelMetadataRepository` | `crates/services/src/models/` | list, get, upsert |
| `UserAliasRepository` | `crates/services/src/models/` | list, get, create, update, delete |
| `AccessRepository` | `crates/services/src/users/` | list, get, create, update |
| `AccessRequestRepository` | `crates/services/src/app_access_requests/` | list, get, create, update |
| `SettingsRepository` | N/A | **NO CHANGES** (global) |

Each `DefaultDbService` impl adds `.filter(Column::TenantId.eq(tenant_id))` to queries and sets `tenant_id` on inserts.
