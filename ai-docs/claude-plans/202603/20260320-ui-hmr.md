# Plan: Next.js basePath `/ui` + Scoped Static Serving + WebSocket Proxy

## Context

The dev proxy (Rust server → Next.js dev server at localhost:3000) has **HMR/WebSocket failures** because `hyper_util::client::legacy::Client` doesn't handle HTTP Upgrade. Additionally, the static file router uses a **fallback pattern** (catches ALL unmatched routes), making the API/UI boundary unclear and proxy behavior unpredictable.

**Solution**: Scope all UI serving explicitly under `/ui/*` using Next.js `basePath` and axum `Router::nest("/ui", ...)`, add SPA fallback for non-asset 404s, and fix the WebSocket proxy for HMR.

**Future context**: Frontend will migrate from Next.js to Vite+React. Solution must be framework-agnostic (single `index.html` SPA fallback, no per-route HTML assumptions).

## Key Technical Insight: basePath Does NOT Change Output Structure

With `basePath: '/ui'` and `output: 'export'`, Next.js:
- Output directory structure is **unchanged**: `out/index.html`, `out/chat/index.html`, `out/_next/...`
- Asset references in HTML **are** prefixed: `/_next/...` → `/ui/_next/...`
- `<Link href="/chat">` renders `<a href="/ui/chat">` (basePath prepended in DOM)
- `usePathname()` returns **without** basePath: browser URL `/ui/chat` → returns `/chat`
- `router.push('/chat')` navigates to `/ui/chat` (basePath auto-prepended)
- `window.location.pathname` returns **with** basePath: `/ui/chat` (raw browser API)
- `redirect('/chat')` redirects to `/ui/chat` (basePath auto-prepended)

This means `include_dir!("../bodhi/out")` stays **unchanged** and `nest("/ui", spa_router)` works naturally.

## Implementation Plan (upstream → downstream)

### Layer 1: `routes_app` — New SPA Router (`spa_router.rs`)

**New file**: `crates/routes_app/src/spa_router.rs`

Create `build_spa_router(dir: &'static Dir<'static>) -> Router` that:
1. Uses `include_dir::Dir` API directly to look up files (bypass `tower_serve_static::ServeDir`)
2. Handles directory requests by appending `index.html`
3. Serves files with correct MIME types via `mime_guess`
4. **SPA fallback**: If file not found AND path has no file extension → serve root `index.html`
5. If file not found AND path has file extension (.js, .css, .png, .woff2, etc.) → return 404

```rust
fn build_spa_router(dir: &'static Dir<'static>) -> Router {
    Router::new().fallback(move |req: Request| async move {
        let path = percent_decode_str(req.uri().path().trim_start_matches('/')).decode_utf8_lossy();
        // Try exact file, then dir/index.html, then SPA fallback
    })
}
```

**Dependencies**: Add `include_dir` and `mime_guess` to `routes_app/Cargo.toml`.

### Layer 2: `routes_app` — WebSocket Proxy (`routes_proxy.rs`)

**File**: `crates/routes_app/src/routes_proxy.rs`

Add WebSocket upgrade detection and forwarding:
1. Check for `Upgrade: websocket` header in incoming request
2. For regular HTTP: use existing `hyper_util::client::legacy::Client` approach
3. For WebSocket: use `hyper::client::conn::http1` with `.with_upgrades()` to establish a raw connection to the backend, forward the upgrade request, then spawn a task to `tokio::io::copy_bidirectional` between client and backend upgraded streams

The proxy handler must also **prepend `/ui`** to forwarded paths since `nest("/ui", proxy)` strips the prefix, but Next.js dev server expects `/ui/*` paths (basePath is active in dev mode too).

```rust
pub fn proxy_router(backend_url: String) -> Router {
    Router::new().fallback(move |req| proxy_handler(req, backend_url.clone()))
}

async fn proxy_handler(req: Request, backend_url: String) -> Response<Body> {
    // nest() strips /ui, but Next.js dev server expects /ui/* paths
    let path = format!("/ui{}", req.uri().path_and_query().map(|x| x.as_str()).unwrap_or(""));

    if is_websocket_upgrade(&req) {
        ws_proxy(req, backend_url, &path).await
    } else {
        http_proxy(req, backend_url, &path).await
    }
}
```

**Dependencies**: May need `hyper` with `http1` feature, `hyper-util` with `tokio` feature (check existing workspace deps).

#### 8C: `handleSmartRedirect` (`src/lib/utils.ts`) — CRITICAL FIX

Backend returns paths with `/ui/` prefix (e.g., `"/ui/login"`, `"http://host/ui/chat"`). With basePath, calling `router.push('/ui/login')` → navigates to `/ui/ui/login` (DOUBLE PREFIX BUG).

Fix: Strip basePath from paths before `router.push()`:
```typescript
export function handleSmartRedirect(location: string, router: { push: (href: string) => void }): void {
    const basePath = '/ui';

    if (location.startsWith('/')) {
        const path = location.startsWith(basePath) ? location.slice(basePath.length) || '/' : location;
        router.push(path);
        return;
    }

    // For full URLs: extract pathname, strip basePath, use router
    try {
        const redirectUrl = new URL(location);
        const currentUrl = new URL(window.location.href);
        if (redirectUrl.protocol === currentUrl.protocol && redirectUrl.host === currentUrl.host) {
            let internalPath = redirectUrl.pathname + redirectUrl.search + redirectUrl.hash;
            if (internalPath.startsWith(basePath)) {
                internalPath = internalPath.slice(basePath.length) || '/';
            }
            router.push(internalPath);
        } else {
            window.location.href = location;
        }
    } catch { window.location.href = location; }
}
```

## Edge Cases

1. **OAuth callback**: Backend constructs `http://host/ui/auth/callback` → browser navigates there → nest matches `/ui/*` → serves SPA → React handles callback. Works.

2. **`docs/` route**: `src/app/docs/` stays at `src/app/docs/`. With basePath, URL is `/ui/docs/`. ServeDir finds `docs/index.html` after nest strips `/ui`. Works.

3. **Favicon/public assets**: `public/bodhi-logo/` copies to `out/bodhi-logo/`. HTML references `/ui/bodhi-logo/` (basePath). Nest strips `/ui/`, finds `bodhi-logo/` in `out/`. Works.

4. **404 page**: Next.js generates `out/404.html`. SPA fallback serves root `index.html` for non-asset paths. The SPA router handles 404 display.

5. **`usePathname()` in docs Navigation**: Already uses `/docs/` without `/ui/` prefix. With basePath, `usePathname()` still returns `/docs/...` (without basePath). Works without changes.
