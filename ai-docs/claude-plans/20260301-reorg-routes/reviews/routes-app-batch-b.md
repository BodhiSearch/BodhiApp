## Summary

**Critical (P0)**: 2 findings -- 7 handlers across `users/routes_users_access_request.rs` and `users/routes_users_info.rs` still use the `Extension<AuthContext>` + `State(state)` legacy pattern instead of `AuthScope`.

**Important (P1)**: 4 findings -- `LoginError` naming inconsistency vs `<Domain>RouteError` convention; 3 `error_meta` attribute regressions in `auth/error.rs` where `args_delegate = false` was dropped, potentially changing error response shapes; 2 unauthenticated handlers in `apps/` using `State(state)` instead of `AuthScope`.

**Minor (P2)**: 4 findings -- N+1 query in apps review handler, `expect()` panics for token extraction in users handlers, pagination field duplication.

**Cosmetic (P3)**: 1 finding -- uncapped page number cast.

### models/ assessment
Consolidation from 4 error enums (`ModelError`, `CreateAliasError`, `PullError`, `MetadataError`) into single `ModelRouteError` is clean. All variants preserved. All handlers use `AuthScope`. Handler naming follows Rails convention (`models_index`, `models_show`, `models_create`, `models_update`, `models_destroy`, `modelfiles_index`). Schema file correctly named `models_api_schemas.rs`. Test files follow Pattern A. No `use super::*`. No issues found.

### apps/ assessment
Renamed from `AppAccessRequestError` to `AppsRouteError` -- consistent with convention. Ownership checks use `auth_scope.tools().get()` and `auth_scope.mcps().get()` correctly in `apps_approve_access_request`. Two unauthenticated handlers use State(state) instead of AuthScope (finding #7). N+1 query pattern in review handler (finding #10).

### auth/ assessment
Error enum not renamed to `AuthRouteError` (finding #3). Three `error_meta` attribute regressions (findings #4-6). All handlers use `AuthScope` + `Session` which is correct for OAuth flow. `generate_pkce` is a public utility function -- acceptable. Test files follow Pattern A.

### users/ assessment
Consolidated from `UserRouteError` + `AccessRequestError` into `UsersRouteError`. `routes_users.rs` correctly uses `AuthScope`. `routes_users_access_request.rs` and `routes_users_info.rs` still use the legacy `Extension<AuthContext>` + `State(state)` pattern (findings #1-2). `AuthScopedUserService` usage is present in `routes_users.rs` via `auth_scope.users()`. Test files follow Pattern A (from handler files) and Pattern B (auth tests from mod.rs).
