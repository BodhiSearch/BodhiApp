## Summary

- **P0 (7 findings)**: All MCP sub-modules (`routes_mcps_auth.rs`, `routes_mcps_oauth.rs`, `routes_mcps_servers.rs`) and `mcps_fetch_tools` use the legacy `Extension<AuthContext>` + `State(state)` pattern instead of `AuthScope`. This is the primary migration gap. The `tokens/` and `toolsets/` CRUD handlers correctly use `AuthScope`.
- **P1 (3 findings)**: `TokenRouteError` has 4 dead variants and 2 semantically incorrect error types (BadRequest for privilege/auth errors that should be Forbidden).
- **P2 (4 findings)**: Business logic leaking into handlers (MCP access filtering, ownership checks), inconsistent router wiring pattern for toolsets.
- **P3 (3 findings)**: Naming convention inconsistencies in test functions, endpoint constants in mod.rs.
