## Purpose

Minimal error infrastructure foundation with zero framework dependencies. Defines the
core error contract (`AppError` trait), HTTP error categories (`ErrorType`), and universal
error types used across the entire crate graph. Exists so lightweight crates
(`llama_server_proc`, `mcp_client`) can participate in structured error handling without
pulling in `axum`, `serde`, or `sea-orm`.

## Architecture Position

- **Depends on**: `errmeta_derive` (proc macro only)
- **Consumed by**: every Rust crate (directly by `llama_server_proc`, `mcp_client`; transitively by others through `services` re-exports)
- `services` re-exports all errmeta types -- downstream crates never need errmeta as a direct dependency

## Boundary: errmeta vs services

**errmeta**: error types with zero framework deps.
**services::shared_objs**: framework-dependent wrappers (`ApiError`, `OpenAIApiError`, `SerdeJsonError`, etc.).

## Non-Obvious Rules

### ErrorType serialization inconsistency
Most variants follow `{name}_error` suffix (e.g., `BadRequest` -> `invalid_request_error`).
Exceptions maintained for backward compatibility:
- `ServiceUnavailable` -> `service_unavailable` (no `_error` suffix)
- `InvalidAppState` -> `invalid_app_state` (no `_error` suffix)

See `src/error_type.rs:1-44` for the full enum with strum serialization overrides.

### impl_error_from! macro
Bridges Rust orphan rule: `ExternalError -> IntermediateWrapper -> ServiceError::Variant`.
Usage: `impl_error_from!(source_type, target_enum::variant, intermediate_type)`.
Used ~20 times across the codebase. Defined in `src/lib.rs:19-28`.

### AppError trait
- `error_type()` -> String (maps to `ErrorType` serialized name)
- `status()` -> u16 (derived from `error_type()` by parsing back to `ErrorType`; defaults to 500)
- `code()` -> String (machine-readable, auto-generated by `errmeta_derive`)
- `args()` -> HashMap<String, String> (context for templated messages)
- Blanket `From<T: AppError> for Box<dyn AppError>` enables trait object usage

See `src/app_error.rs` for trait definition.

### RwLockReadError has manual AppError impl
Its single-field struct layout does not fit the enum-based `ErrorMeta` pattern.
See `src/rwlock_error.rs`.

### IoError convenience constructors
`IoError::file_read(err, path)`, `dir_create()`, `file_write()`, `file_delete()`, `with_path()`.
All variants capture both `std::io::Error` source and filesystem path. All map to 500.
See `src/io_error.rs:54-88`.

### args() calls format!("{}", field) on all named fields
Fields must implement `Display`. `Option<String>` does not work -- need manual `AppError` impl.
This is a `errmeta_derive` behavior that surfaces here.

## Error Flow Through the System

```
errmeta:           AppError trait, ErrorType, IoError, EntityError
                        |
errmeta_derive:    #[derive(ErrorMeta)] generates AppError impl
                        |
services:          Domain error enums (each implementing AppError via ErrorMeta)
                        |
services:          ApiError (From<T: AppError>) -> OpenAIApiError -> JSON HTTP response
```
